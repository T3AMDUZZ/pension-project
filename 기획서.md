# 펜션 예약 웹사이트 기획서

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 프로젝트명 | 펜션 예약 관리 웹사이트 |
| 프로젝트 유형 | 펜션 홍보 + 예약 관리 시스템 |
| 대상 사용자 | 일반 고객 (예약자) / 펜션 관리자 (운영자) |
| 결제 방식 | 무통장입금 / 현장결제 |
| 반응형 | 모바일 우선 반응형 디자인 |
| 레퍼런스 | 여기어때, 펜션나라 등 숙박 예약 플랫폼 |

## 3. 사용자 페이지 상세 기능

### 3.1 메인 페이지

| 영역 | 설명 |
|------|------|
| 히어로 배너 | 펜션 대표 이미지 슬라이드 (3~5장) |
| 빠른 예약 | 체크인/체크아웃 날짜 선택 → 빈 객실 검색 |
| 객실 미리보기 | 대표 객실 3~4개 카드형 노출 (이미지, 가격, 인원) |
| 펜션 소개 | 간략한 소개 문구 + 대표 이미지 |
| 공지사항 | 최근 공지 2~3건 |
| 푸터 | 펜션 주소, 연락처, 계좌 정보, 사업자 정보 |

### 3.2 객실 안내

#### 객실 목록
- 카드형 레이아웃 (이미지 + 객실명 + 기준인원 + 가격)
- 필터: 인원수별, 가격순 정렬
- 예약 가능 여부 뱃지 표시

#### 객실 상세
| 항목 | 설명 |
|------|------|
| 이미지 갤러리 | 객실 이미지 슬라이드 (5~10장) |
| 기본 정보 | 객실명, 기준/최대 인원, 객실 크기, 객실 유형 |
| 가격 정보 | 비수기/성수기, 주중/주말 가격표 |
| 편의시설 | 아이콘 + 텍스트 (에어컨, TV, 바베큐, 와이파이 등) |
| 객실 소개 | 상세 설명 텍스트 |
| 예약 달력 | 해당 객실 예약 가능 날짜 캘린더 |
| 예약 버튼 | "예약하기" CTA 버튼 |

### 3.3 예약 기능

#### 예약 프로세스 (3단계)

```
[STEP 1] 날짜 선택        →  [STEP 2] 정보 입력       →  [STEP 3] 예약 완료
- 체크인/체크아웃 날짜       - 예약자 이름              - 예약 번호 발급
- 객실 선택                 - 연락처 (휴대폰)          - 예약 정보 요약
- 인원 수 입력              - 차량번호 (선택)          - 입금 계좌 안내
                            - 결제 방식 선택           - 카카오톡/SMS 안내 발송
                              (무통장입금/현장결제)
                            - 요청사항 (선택)
```

#### 예약 달력 상세
| 항목 | 설명 |
|------|------|
| 표시 방식 | 월간 달력 뷰 |
| 예약 가능 | 초록색 또는 기본색 |
| 예약 마감 | 빨간색 또는 회색 처리 |
| 날짜 선택 | 체크인/체크아웃 범위 선택 (드래그 또는 클릭) |
| 가격 표시 | 각 날짜 칸에 해당일 가격 표시 |
| 최소 숙박 | 성수기 최소 2박 등 정책 반영 |

#### 예약 조회
- 예약번호 + 연락처로 조회
- 예약 상태 확인 (대기중 / 입금확인 / 확정 / 취소)
- 예약 취소 요청 가능

### 3.4 펜션 소개

| 페이지 | 내용 |
|--------|------|
| 인사말 | 펜션 소개 텍스트 + 대표 이미지 |
| 오시는 길 | 카카오맵 또는 네이버맵 연동, 주소, 교통편 안내 |
| 주변 관광지 | 이미지 + 설명 카드 리스트 |

### 3.5 공지사항 / 이용안내
- 공지사항 목록 (페이징)
- 이용안내 페이지 (입퇴실 시간, 환불 규정, 바베큐 이용 등)

## 4. 관리자 페이지 상세 기능

### 4.1 대시보드

| 항목 | 설명 |
|------|------|
| 오늘의 현황 | 오늘 체크인 / 체크아웃 / 투숙중 건수 |
| 신규 예약 | 최근 신규 예약 리스트 (빠른 확인) |
| 입금 대기 | 무통장입금 대기 건수 (강조 표시) |
| 이번 달 매출 | 월간 매출 요약 |
| 객실 현황 | 오늘 기준 객실별 상태 (빈방/투숙중/청소중) |
| 월간 달력 | 예약 현황 캘린더 (객실별 색상 구분) |

### 4.2 예약 관리

#### 예약 목록
| 기능 | 설명 |
|------|------|
| 목록 표시 | 테이블 형태 (예약번호, 예약자, 객실, 날짜, 상태, 결제방식) |
| 검색/필터 | 기간별, 객실별, 상태별, 예약자명/연락처 검색 |
| 상태 관리 | 예약대기 → 입금확인 → 예약확정 → 체크인 → 체크아웃 |
| 예약 취소 | 관리자 직접 취소 처리 |
| 메모 | 예약건별 관리자 메모 기능 |

#### 예약 달력 뷰
- 월간 캘린더에 객실별 예약 현황 바(bar) 표시
- 색상으로 예약 상태 구분
- 달력에서 직접 예약 등록 가능 (관리자 수동 예약)
- 드래그로 날짜 변경 (선택적)

### 4.3 객실 관리

| 기능 | 설명 |
|------|------|
| 객실 목록 | 등록된 객실 리스트 (노출순서 변경 가능) |
| 객실 등록/수정 | 객실명, 소개, 기준/최대 인원, 크기, 편의시설 체크 |
| 이미지 관리 | 다중 이미지 업로드, 순서 변경, 대표 이미지 지정 |
| 노출 설정 | 객실 노출/숨김 처리 |

### 4.4 요금 관리

```
요금 구조:
┌─────────────┬──────────┬──────────┐
│             │  주중     │  주말     │
├─────────────┼──────────┼──────────┤
│ 비수기       │ 기본가    │ 기본가+α  │
│ 준성수기     │ +β       │ +β+α     │
│ 성수기       │ +γ       │ +γ+α     │
│ 극성수기     │ +δ       │ +δ+α     │
└─────────────┴──────────┴──────────┘
+ 인원 추가 요금 (기준인원 초과 시)
```

| 기능 | 설명 |
|------|------|
| 시즌 설정 | 날짜 범위로 시즌 구분 (비수기/준성수기/성수기/극성수기) |
| 객실별 요금 | 객실별로 시즌×요일 조합 가격 입력 |
| 추가 인원 요금 | 기준인원 초과 시 1인당 추가 금액 |
| 특가 설정 | 특정 날짜 직접 가격 오버라이드 |

### 4.5 공지사항 관리
- 공지사항 CRUD (작성, 수정, 삭제)
- 에디터 (텍스트 + 이미지 첨부)

### 4.6 기본 설정

| 항목 | 설명 |
|------|------|
| 펜션 정보 | 펜션명, 주소, 연락처, 소개글, 대표이미지 |
| 입금 계좌 | 은행명, 계좌번호, 예금주 |
| 예약 정책 | 최소 숙박일수, 예약 가능 기간 (몇일 전까지), 환불 규정 |
| 관리자 계정 | 아이디/비밀번호 변경 |

## 5. 예약 상태 흐름 (Status Flow)

```
[예약신청] ──→ [입금대기] ──→ [입금확인] ──→ [예약확정] ──→ [체크인] ──→ [체크아웃]
    │              │
    │              ├──→ [기한초과 자동취소]
    │              │
    └──→ [현장결제 선택 시] ──→ [예약확정] ──→ [체크인] ──→ [체크아웃]
                                    │
                                    └──→ [노쇼]

※ 관리자는 어떤 상태에서든 [취소] 처리 가능
```

## 6. 알림 기능

| 시점 | 방법 | 내용 |
|------|------|------|
| 예약 신청 시 | SMS/카카오 알림톡 | 예약 접수 안내 + 입금 계좌 (무통장 선택 시) |
| 입금 확인 시 | SMS/카카오 알림톡 | 예약 확정 안내 |
| 체크인 전날 | SMS/카카오 알림톡 | 체크인 안내 (시간, 위치, 주의사항) |
| 관리자 | 웹 알림 / 이메일 | 신규 예약 알림 |

> ※ 알림톡 연동은 별도 API 비용 발생 (선택사항, 1건당 약 8~15원)

## 8. 기술 스택 (권장)

| 구분 | 기술 | 비고 |
|------|------|------|
| 프론트엔드 | Next.js (React) | SSR/SSG 지원, SEO 최적화 |
| 스타일링 | Tailwind CSS | 빠른 반응형 개발 |
| 백엔드 | Next.js API Routes | 별도 서버 불필요 |
| 데이터베이스 | PostgreSQL (Supabase) | 무료 플랜 활용 가능 |
| 이미지 저장 | Supabase Storage / Cloudflare R2 | 저렴한 이미지 호스팅 |
| 배포 | Vercel | 무료 플랜 또는 월 $20 |
| 알림 | 카카오 알림톡 API (선택) | NHN 클라우드 등 |
| 지도 | 카카오맵 API | 무료 |

### 예상 운영 비용

| 항목 | 월 비용 |
|------|---------|
| Vercel (호스팅) | 무료 ~ $20 |
| Supabase (DB) | 무료 ~ $25 |

## 10. 데이터 모델 (주요 테이블)

```
[rooms] 객실
├── id, name, description, type
├── base_capacity, max_capacity, size
├── amenities (JSON)
├── images (JSON)
├── sort_order, is_active
└── created_at, updated_at

[seasons] 시즌
├── id, name (비수기/성수기 등)
├── start_date, end_date
└── min_nights (최소 숙박일)

[room_prices] 객실 요금
├── id, room_id, season_id
├── weekday_price, weekend_price
└── extra_person_price

[special_prices] 특가
├── id, room_id, date
└── price

[reservations] 예약
├── id, reservation_number
├── room_id, guest_name, phone
├── check_in, check_out, guests_count
├── payment_method (무통장/현장)
├── total_price
├── status (대기/입금확인/확정/체크인/체크아웃/취소/노쇼)
├── vehicle_number, memo, admin_memo
└── created_at, updated_at

[notices] 공지사항
├── id, title, content
├── is_pinned
└── created_at

[settings] 설정
├── key, value
└── (펜션정보, 계좌정보, 정책 등)

[admin_users] 관리자
├── id, username, password_hash
└── created_at
```
